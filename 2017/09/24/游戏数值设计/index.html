游戏数值设计

发表于 2017-09-24

职业基础属性设计

基础属性

一级属性：不直接参与战斗的人物属性，可以通过升级或加点的方式获得。二级属性：一般情况下自身无法成长，需要依附一级属性成长或是装备或其他系统加成的属性。
常见MMORPG的一级属性有： 

- 力量（物理攻击力，物理防御力，生命值）
- 智力（魔法攻击力，魔法防御力，魔法值）
- 敏捷（命中，闪避）
- 精神（暴击，抗暴击）
- 体质（生命值，物防，魔放）

二级属性：
生命值,魔法值,物理攻击,物理防御,魔法攻击,魔法防御,命中,闪避,暴击,抗暴击

一级属性和二级属性一般通过转换系数表来转换。下面是一个常见的转换系数表。

属性 物攻 物防 魔攻 魔防 生命值 魔法值 命中 闪避 暴击 抗暴击
力量 1 0.4 2
智力 1.1 0.45 1
敏捷 0.02 0.02
精神 0.02 0.02
体质 0.6 0.6 16

如果想让游戏各属性对职业的价值不同，这个系数表可以根据职业而变化。

标准人和职业定位

在设计属性之前，首先需要设定一个标准人的属性，然后对比标准人的属性成长来设定各个职业属性。
在标准人的基础上设计职业，比如战士应该血高防厚，法师应该攻高血少。职业的设定不单单体现在属性上，还有技能等其他方向。用来衡量不同职业能力差异表如下。

能力 生命值 攻击力 输出能力 回复能力 控制能力 辅助能力 综合评分 评级 分值
战士 A A B C C C 650 A 120
法师 C A A C A B 670 B 110
刺客 B B A C A B 670 C 100
牧师 C C C A B A 650

通过打分对各职业设定一定倾向。比如A为标准的120%~110%，B为标准110%~100%，C为标准的100%~90%，然后得到各职业生命值数据。最终通过调整实现平衡。

DPS、有效生命和角色强度

  • 决定DPS的主要来自：
    1.普攻的数值
    2.普攻的速度，对DPS影响很大。

  • 有效生命值：是指防御方在综合防御，闪避，暴击等所有战斗因素后，得到的防御方在上述条件下的最终生命数值。计算公式为：
    $$有效生命=\frac{生命值}{(1-伤害减免率)\cdot (1-闪避率)}$$

战斗公式设计

战斗流程

  • 一次典型的近战模式攻击：前摇(0.5s)→完成结算（瞬间）→后摇(0.5s)
    CD计算方式有三种，其不同如下表：
    假设冷却时间为1s
结算时间 单位CD时间 每分钟攻击次数
前摇前 =0.5+0.5+1 =60/2=30
前摇后 =0.5+1 =60/1.5=40
后摇后 =1 =60/1=60
  • 一次典型的远程模式攻击：法师使用火球（飞行时间不可控），一般远程攻击在触碰到目标时候才计算战斗公式。
  • 引入公共冷却时间，给所有技能加入一段公共冷却时间，更好控制技能输出量。
  • 攻速：一般即时游戏用来衡量攻击快慢，回合制游戏攻速（也叫先手值或敏捷）用来决定出手的先后顺序。
    一般而言，这里的攻速是前面整个攻击环节所需时间换算得来。比如0.5s/1次的攻击也就是2次/秒。大部分游戏会将攻击速度与武器挂钩。

战斗公式流程

战斗公式的流程主要有两种，一中是圆桌算法，一种是瀑布算法。

  1. 瀑布算法：一步一步判断
    攻击开始：判定是否命中，未命中则本次攻击没有伤害，命中判定本次是否暴击，没有暴击则结算为普攻，暴击则进入暴击伤害计算公式。
    流程比较清晰，但判断结果类型多了后，控制数值比较困难。
  2. 圆桌理论，高优先级的概率和达到100%，就不会出现普攻，这样就把低优先级的结挤出了桌面。
    圆桌理论的判定：在某一属性极端增大的情况下，会出现不均衡的情况。如未命中率增大到85%，暴击增大到20%，这样突破了圆桌的上限，按100随机数法，1~85代表未命中，86~105代表暴击，然而实际取1~100的随机数。这样暴击真实概率为15%，并非理论20%。
  3. 比较：圆桌算法在底层被挤出”圆桌”前，各高优先级不会产生关联影响，概率更为合理。但出现挤出去的情况下，有时会产生异常打的影响。
    逐步判断则是优先级高的情况会严重影响优先级低的情况，并且逐级影响，层级越多情况越复杂。但这个情况按比例稀释，不会出现特别极端情况。

一级属性和二级属性

一般来说，如果游戏有加点或者洗点的系统（或装备有很大属性差异），需要一级属性。反之不要一级属性也可。

闪避公式

  1. 命中先减去闪避，由差距大小决定命中概率（少）

    1. 闪避大于等于命中时，命中等于保底命中下限30%（假定），由于闪避投放远小于命中，这种情况会尽量避免发生。

    2. 命中大于闪避，计算命中差值。基本思路是60%的基础命中率，之后差值每增加1点，命中率增加0.5%，最大到95%。

      这种方法对游戏后期投放不利，点插值带来收益永远恒定。

  2. 命中和闪避进行除法公式计算。
    $$命中率=\frac{命中}{命中+闪避}$$
    这种方法使得命中和闪避在前期加点效果非常明显，玩家很容易从数值提升感受到战斗效果提升，而对属性的需求变的强烈，进而促进消费。命中和闪避到达一定数值时，开始产生边际效应，但也不会完全失效，玩家依然有提升意义。大部分游戏基于这个公式设计。

暴击公式

  1. 暴击减去抗暴击，然后差值转换为暴击率。
    在暴击率设计中引入职业影响参数。暴击率=职业基础暴击率+暴击值转换暴击率。早期游戏设计中是希望职业有区分，现在游戏基本不会给职业加固定暴击率了。暴击值转换暴击率的公式为：
    $$暴击率=K1\cdot \frac{暴击值}{暴击值+K2\cdot 人物等级+K3}$$

    其中K1，K2，K3为三个系数，K1代表最终暴击率的理论极限。这个公式使得前期加暴击效果非常明显，但随着等级的提升，玩家需要更多的暴击值来减少等级对暴击率所带来的影响。

  2. 暴击和抗暴击进行除法，类似于闪避公式。

伤害公式

  1. 减法公式
    伤害=攻击-防御。此外目前的设定多为，先判断防御是否大于攻击的1%~10%，如果大于则在攻击的1%~10%内浮动，保证就算不破防，依然可以发挥攻击1%~10%的效果。如果小于则按减法公式计算。减法公式的好处是数值敏感，攻击的增加会直接反映在伤害上。
  2. 乘除法公式
    1. 通过护甲计算出减免系数
      伤害=攻击×（1-伤害减免百分比）
      伤害减免百分比=护甲/（护甲+人物等级×K1+K2）
      人物等级乘以K1是为了让随着等级的成长，是的护甲转换伤害减免百分比变相削弱，让玩家追求更高等级装备。K2是为了控制防御转换成伤害减免百分比的一个密度分布。
      2.通过攻击和防御一起计算出伤害
      伤害=攻击×攻击/（攻击+防御）
      这一公式强调了攻击的价值，目前的变种为：
      伤害=（攻击×K1+K2）/(攻击+防御×K3+K4)

暴伤公式

暴击伤害=普通伤害×（1+伤害暴击系数+特殊效果系数）+暴击后附加伤害
早年游戏伤害暴击系数多为1，近年多为0.5，这样为150%爆伤。

技能设计

技能基本上可以分为四大类：伤害类，控制类，BUFF类技能，探险类技能。

  1. 伤害类技能，设计思路通常以普通攻击为标准，技能在这个基础上去做加值。一般来说输出的技能有三种。第一种攻击增幅一般，CD较短，这个技能一般会将DPS提升10%~100%。这个技能是为增加输出能力的技能，往往也是性价比最高的技能。第二种大幅增加输出，但CD会很长，合理运用这个技能的释放时机可以在短时间内秒杀更强大的怪物。给玩家的体验是攻击造成的伤害非常可观，怪物血量明显减少。第三种是范围伤害技能，一般DPS在普攻的60%~120%。
  2. 控制类技能。分为硬控和软控。软控的价值衡量与之前的范围攻击技能类似，参照普攻折算出该技能的DPS。硬控的核心元素为晕眩的持续时间，另外一个是冷却时间。衡量硬控可以通过控制技能CD时间内平均DPS决定。
  3. BUFF类技能。一般来说，对于加血技能，集合该系数与同等级DPS系数得出加血应有的数值。比如牧师职业治疗能力系数为2，DPS对应系数为0.7，那么牧师的治疗量应为目前DPS的1.4倍，使得击杀牧师很难做到。
  4. 探险类技能。一般与战斗没有联系，不对这类技能进行严密的设计。

装备设计

装备及周边是人物战力的重要组成，一般超过50%，也是最受玩家追捧的游戏元素。

  1. 装备属性设计：分为主属性与其它属性。主属性最大意义是代表这件装备的倾向性，但未必是装备中拥有最大价值的属性。

游戏内掉落方式

  1. 计数随机。道具掉落与怪物被击杀次数相关，这种设计在单机游戏中常见，市面上大多数游戏都不采用这种方法了。
  2. 逐个百分比掉落。假定有A、B、C、D这四个道具，那么先计算A是否掉落，然后计算A掉落数量，再计算B是否掉落以及掉落的数量，直到所有道具循环完毕。这种设计易于理解，但无法控制掉落组合的精确数量。
  3. 权重掉落组式掉落。通过母集和子集，可以设计丰富的掉落组合，并且不管是任何系统关联的掉落都可以只用一个母集的编号对应。缺点是母集与子集的关联性太强，容易出数据问题，对数值本身要求高。
  4. 进阶掉落。进阶掉落是在权重掉落组式掉落的基础上做进化的版本，核心思想一样，多了一步前置判断，在每次母集掉落之前判断一次母集是否满足一定条件并产生进阶掉落。这是目前很多游戏采用的做法。
  5. 木桶原理掉落。让你最短缺的道具获得更高的掉率率。首先计算 空缺量=预期量-现有量，空缺量最小为10，然后根据空缺量做出空缺率，空缺率=空缺量/预期量，在根据空缺率算出系数加成。
